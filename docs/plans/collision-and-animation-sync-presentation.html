Collision in Entity Definitions

// packages/shared/src/champions/definitions/warrior.ts

const collision: CircleCollision = {
  type: 'circle',
  radius: 14,
  offset: { x: 0, y: 4 }
};

export const WarriorDefinition = {
  id: 'warrior',
  name: 'Kael',
  // ... existing stats ...

  // Collision lives WITH the entity!
  collision,
  animations,
};
// packages/shared/src/types/minions.ts

export const DEFAULT_MINION_STATS = {
  melee: {
    health: 477,
    // ... existing stats ...
    collision: { type: 'circle', radius: 10 }
  },
  caster: {
    collision: { type: 'circle', radius: 8 }
  }
};

Key Points:

  • Colocated - Data lives with entity definition
  • Single file - All entity data in one place
  • Easy to add - New entities just add collision field
  • No sync issues - No separate registry to maintain

System Architecture

Colocated Data Architecture

packages/shared/src/

types/
collision.ts - Shape types
animation.ts - Timing types
minions.ts - + collision/anim
structures.ts - + collision
champions/definitions/
warrior.ts - + collision/anim
magnus.ts - + collision/anim
elara.ts - + collision/anim
... each champion owns its data
Entity definitions contain collision shapes and animation keyframes

Server

AnimationScheduler
Reads keyframes from definitions
CollisionSystem
Uses entity.collision shape
ServerChampion
Has collision & animations

Client

SpriteAnimator
Plays animations from definitions
EntityRenderer
Debug collision visualization
StateManager
Handles events from server

Implementation Phases

1

Foundation

  • + Create types/collision.ts
  • + Create types/animation.ts
  • + Add animation timing utils
  • + Export from shared (ts + js!)
2

Server Collision

  • + Shape-aware collision functions
  • + Update CollisionSystem
  • + Add collision to champion defs
  • + Add collision to minion/jungle
3

Animation Scheduler

  • + Implement AnimationScheduler
  • + Integrate with GameRoom
  • + Action execution logic
  • + Cancellation/interrupt handling
4

Attack Sync

  • + Update BASIC_ATTACK event
  • + Modify performBasicAttack()
  • + Add attack anims to champion defs
  • + Minion attack animations
5

Ability Sync

  • + Update ABILITY_CAST event
  • + Modify ability executor
  • + Add ability anims to champion defs
  • + Client keyframe handling
6

Polish

  • + Remove legacy code
  • + Performance profiling
  • + Edge case testing
  • + Documentation update

Key Benefits

Gameplay Feel

  • • Attacks feel satisfying - damage syncs with visual
  • • Abilities feel responsive - projectiles spawn at cast point
  • • Collision feels accurate - hitboxes match sprites

Technical Quality

  • • Server remains authoritative - no client exploits
  • • Shared metadata - single source of truth
  • • Easy to add new entities with proper config
